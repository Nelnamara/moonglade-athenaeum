<mg-model-picker> — standalone verification

Isolates the shared web component (Option-A pilot) from the Loom, so a picker bug and a React-bridge bug can't be confused. Serve the gallery, then open this at /static/mg-model-picker.html on the same machine (it calls the localhost-gated /api/model-search).

picked: (none)
  1. Cards should browse on open (popular models, no typing).
  2. Typing filters; hovering a card (after a short pause -- D-11) shows a preview (cover · likes · base · Official · description).
  3. Clicking a card marks it selected and updates picked: below via the mg-pick event.
selected: (none)
  1. Clicking a card toggles it (add/remove), unlike the single Model picker above.
  2. Each pick resolves its real version_id/lora_base_type/trigger_words via /api/model-version -- selected: below should show a version_id shortly after picking, not immediately.
  3. Multiple LoRAs can be selected at once; clicking a selected one again removes it.
