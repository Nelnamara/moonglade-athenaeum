<mg-generate-drawer> — standalone verification

Isolates the shared Generate form (Option-A component #3, Video tab first) from both hosts, so a component bug and a mount-bridge bug can't be confused. Serve the gallery, then open this at /static/mg-generate-drawer.html on the same machine (it calls the localhost-gated /api/price / /api/loom/generate). Slot picks are serviced by <mg-gallery-picker> via the mg-pick-request event — exactly how a real host wires it.

events: (none yet)
  1. Mode buttons reshape the slots: First frame = 1, First + last = start/end, Multi-ref = up to 9 with + add.
  2. Clicking a slot opens the shared gallery picker (the mg-pick-request seam); hovering a filled slot shows the floating preview.
  3. With a slot filled, the cost line calls the real /api/price — a matching free card shows the green 🎫 line; V4.0 full shows the ⚠ warn.
  4. In Multi-ref, typing @image1 then a space becomes a thumbnail chip; the payload log shows it serialized back to text.
  5. Generate video spends real credits/cards — every submit/result/error lands in the event log below. Don't click it casually.
