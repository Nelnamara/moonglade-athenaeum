## A. Confirmed defects (10 unique) — none previously tracked, ALL FIXED

### Loom V2 / today's code
1. **Deep Focus modal edits visibly revert — the modal renders a stale snapshot.** `setDeepFocus(e)`
   stores the entry *object* at double-click time; every edit (dfPatch → immutable patchCard) creates a
   NEW card, so the modal's controlled inputs snap back each keystroke while the board behind it shows
   the change. Title/duration typing is eaten, mode chips never move, picked frames never appear. Only
   open/Escape/close/Select were ever verified — the editing path never was.
   Fix: derive the live entry from `entries` each render (or store only ids). `master-storyboard.jsx:1158`. **broken**
2. **No way to deselect a shot in V2 → draft mode is one-way.** Every surface only ever sets selShot;
   nothing in V2 clears it. The whole draft UI (route picker, finished-result attach flow) gates on `!sel`,
   so clicking any shot while a draft renders makes the attach flow unreachable (non-destructive recovery:
   switch projects and back; the video itself is safe in the gallery and attachable via "Use an existing
   video"). Fix: board-card click toggles (`setSelShot(id===selShot?null:id)`) + an explicit unbind button.
   `master-storyboard.jsx:740`. **degraded** (found independently by two agents)
3. **generateShot's credit confirm fails OPEN.** `priceShot` swallows all errors (`catch { return null }`)
   and the guard `if (pr && !pr.free && pr.cost != null && !confirm(...))` short-circuits on null — any
   /api/price failure (which itself returns HTTP 200 + cost:null on ANY server exception) submits a paid
   video generation with no confirmation. batchGenerate has the milder variant (null prices count as 0 in
   its total). Fix: fail closed — null price ⇒ still confirm. `master-storyboard.jsx:1420`. **degraded**
4. **Attaching a new clip keeps the shot's OLD trimIn/trimOut** — a stale trimOut past the new clip's end
   makes SequencePlayer hang on that clip forever (no onEnded fallback); stale trimIn skips it instantly;
   export's ffmpeg gets out-points past EOF. Applies to the draft attach, pollShot's completion write, AND
   useExistingVideo. Fix: reset `trimIn:0, trimOut:null` whenever resultMid changes + add an onEnded
   fallback to SequencePlayer. `master-storyboard.jsx:886`. **degraded**
5. **ShotPreview's `playing` state goes stale when the selected shot changes** (V2 timeline reuses one
   instance, no key): src swap silently pauses, button sticks on ⏸ (two clicks needed), hover-scrub stays
   disabled, and `dur` is the old clip's until new metadata — a 404'd video lets wrong-duration trims land
   on the new shot. Fix: `key={sel.c.id}` on the timeline instance + `.catch(()=>{})` on play().
   `master-storyboard.jsx:783`. **degraded**
6. **`__draft__` generation state is never cleared** — a finished draft from project A resurfaces in
   project B's drawer offering to attach the wrong video (applies to genState + genImg/genEdit/genRef
   states). Fix: clear the `__draft__` keys on project switch / V2 unmount. `master-storyboard.jsx:809`. **polish**
7. **Trim handles draggable while playing** — startDrag/onMove lack the while-playing guard scrub/mouseLeave
   got in the same commit; out-handle drags trigger mid-drag pause-and-rewind. Fix: pause on startDrag.
   `master-storyboard.jsx:1956`. **polish**
8. **Deleting a storyboard races the 600ms autosave debounce** — a pending save timer can fire during
   deleteProject's awaits and re-create the key after sDel (resurrected board). Fix: `clearTimeout(saveTimer.current)`
   at the top of deleteProject. `master-storyboard.jsx:1289`. **polish**
