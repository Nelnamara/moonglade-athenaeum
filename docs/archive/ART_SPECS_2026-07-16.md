## Global rules (read once)

**Palette** — match these so art sits in the UI, not on top of it:
| Token | Hex | Role |
|---|---|---|
| base | `#0c0a1c` | page background (near-black violet) |
| mantle | `#0a0818` | header/panels (darkest) |
| lavender | `#b692e6` | **primary accent** — the "leads" color |
| mauve | `#c4a6f0` | soft accent |
| purple-deep | `#33236d` | deep armor violet |
| purple-bright | `#643aac` | brighter violet |
| gold | `#d4af37` | rare filigree/trim (use sparingly) |
| emerald | `#4fc99a` | the "magic" glow (gems, Void-light) |

**Theme:** Moonglade / night-elf / a-library-against-the-Void. Moonlit, arcane, deep
violet with emerald magic-glow and rare gold. Dark and moody, not bright.

**Format:** JPG for full backdrops (no transparency needed). **PNG for anything with
transparency** (logo, favicon, badges).

**Transparency gotcha (important):** PixAI/Mio output has *fake, painted* transparency —
it won't give a real alpha channel. For any cut-out asset, **generate on a solid pure-green
`#00FF00` background** and say so in the prompt ("subject centered on a flat chroma-green
#00FF00 background, no shadow"). We key the green out cleanly afterward. A "transparent
background" prompt gives a checkerboard *painted into* the image — unusable.

## 1. Header banner — LIVE slot, ready now
**Drop at:** `pixai_backup/branding/banner.png` → appears instantly, no restart.
**It must be named `banner.png`** — a `.jpg` is silently ignored: the slot never lights up,
the hardcoded `/branding/banner.png` 404s, and the `<img>` deletes itself on error, so you
get no art and no clue why. Convert to PNG.
- **Master size:** **2048 × 512 px or wider** (matching the banner-v2 spec in
  `docs/ART_PROMPTS.md`). The code enforces no ratio at all — `#brand-banner` is
  `object-fit: cover`, so it crops whatever you give it. (The old **1920 × 480** ≈ 4:1
  figure came from the retired ~58 px header strip and no longer describes anything.)
- **How it's shown:** stretched to cover a **hero header 150–300 px tall**
  (`clamp(150px, 22vw, 300px)`) that slides up to a 62 px slim nav bar as you scroll,
  **cropped at `object-position: center 32%`** (the upper-middle, not the vertical center),
  at **full opacity**, with a **vertical darkening scrim** over the lower half so the nav
  stays readable. (The 16% opacity + left→right fade still in the CSS is the *no-banner*
  base state — it never applies once real art is present.)
- **Compose for that:** the **whole frame shows**, so compose the whole frame — keep the
  hero element around the **upper-middle** band, since the bottom is scrimmed and the top
  crops away first as the header collapses. Full opacity means fine detail survives now,
  but it still sits behind the wordmark and nav, so favor atmosphere over busy subject
  detail. Think "moonlit Nelnamara silhouette / Moonglade canopy," not a busy scene. Deep
  violet with an emerald glow accent is ideal.
- No transparency needed (it's a backdrop) — but the file is still PNG, per the name rule.

## 2. Favicon / browser-tab icon — LIVE slot, hook shipped
**Drop at:** `pixai_backup/branding/favicon.png` → picked up by the `<link rel="icon">` every
page already ships. An inline SVG fallback (the lavender "M" tile with the gold dot) covers
its absence, so nothing needs wiring on request.
- **Master:** **512 × 512 px PNG**, transparent (green-screen method).
- The eclipse-moon or the "M" sigil. Must read at **16 px** — bold, one clear shape, high
  contrast. No text, no fine detail.

## 3. Logo mark — superseded by the marks system (`logo.png` is now just the fallback)
- **Master:** **512 × 512 px PNG**, transparent, square.
- Displays at **42 px** in the header, so same rule: one bold sigil (crescent + gem, or a
  stylized M with the gold dot), reads tiny. Currently a lavender rounded tile with a gold
  dot and an eclipse sweep — a real mark should keep that eclipse/gold-dot DNA.
- **A full marks system ships now:** cut marks live in `branding/marks/` (`marks.json`), are
  chosen in Panel > Branding via `/api/branding`, and get injected into every page as
  `mark_url`/`mark_anim`/`mark_kind`. A single `branding/logo.png` is only the legacy
  fallback used when no cut marks exist.

## 4. Windows launch icon — automated (your double-click shortcut)
- **Master:** **256 × 256 px PNG**, transparent. Same sigil as the logo mark works.
- No manual conversion step: marks land in `branding/marks/` as PNG **+ a multi-res `.ico`**
  (16/32/48/256), and Panel > Branding's **Set launcher icon** POSTs `/api/branding/shortcut`,
  which writes the Desktop `.lnk` pointed at that `.ico` itself.

## Art for shipped systems

## 5. Skins (theme packs) — SHIPPED (v1.11.0)
**Needs no art.** The five shipped skins (Moonglade, Nightfallen, Moonlit Silver, Embercourt,
Verdant Grove) are **accent-palette swaps only** — `html[data-skin=…]` token overrides applied
via `/api/skin`. The "each skin = one banner + an accent swap" model specced here was never
built: there is exactly **one** banner slot (#1), shared by every skin.

## 6. Achievement badges (the "IT'S OVER 9000" system) — SHIPPED (v1.11.0)
- **Master:** **2000 × 2000 px PNG**, transparent (green-screen), **circular emblem** style —
  WoW-achievement look: ring, icon in the center, glow. One per achievement, 57 in all
  (~300 MB of masters). **Don't generate at 256** — the app lazily derives its own ~256 px
  thumbs into `branding/_thumbs` and serves them at `/badge-thumb/<id>.png`, so a full Trophy
  Hall open never pulls the masters. The masters stay the source of truth.
- **The ring is tier-encoded, not gold for all:** common → weathered silver · rare → polished
  blue-steel · epic → ornate amethyst-and-gold · legendary → radiant royal gold. See
  `docs/ART_PROMPTS.md` for the badge prompt template.
- Rendered at **46 px** on grid tiles, **38 px** in the Recent rail, and **100 px** on the
  unlock toast. Design to "reads clearly at 64 px" and it lands at all three.
