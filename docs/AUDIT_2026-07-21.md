- **The single biggest miss was a regression, not a gap.** The gallery Video tab's lost Job Tracker registration (§1) was found by *one* refuter, not by any lens. The orphaned-surfaces lens audited component APIs **globally** ("does anything, anywhere, listen for `mg-cost`?") and got the inverse result — it flagged an optional presentational hook with no consequence and missed a real dropped capability. The right question for a partial migration is **per-mount**: for each host of a shared component, which events does *that host* consume, and what did it lose relative to its pre-migration code? Run that matrix and gallery-1-of-14 vs Loom-14-of-14 is the first row.
- **A second cheap heuristic nobody applied:** for any function deleted in a migration commit, grep the surviving comments that *name* it. `Gen.videoGenerate` is gone from the code and still cited as live precedent in the Loom.
- **Reachability was asked of defects but almost never of remedies or of the codebase as a whole.** No lens enumerated `@app.route` endpoints against `url_for`/literal `fetch()` references — one script closes "reachable from the UI?" for all 86 routes at once. (It comes back clean, which is a *stronger* result than "I checked the two I already knew about.") Conversely, nobody asked the inverse question — **which LOCALHOST-gated controls are rendered to every LOGIN session?** There are three: ↑ Import (`:3911`, inside the always-true `{% if is_local %}`), "Set launcher icon" (`:7319`, on a page that passes no locality flag at all), and every destructive Panel action button — each of which walks a LAN user through a confirm dialog before 403ing.
- **Class-level sweeps were done by anecdote.** `--variant` was found by reading an archived HTML dashboard; a five-line AST pass over all 115 argparse dests settles the whole class and proves it's the *only* dead flag. Same for the `not in` mirror of the substring idiom (93 assertions — clean, but "I checked and it's clean" was owed and never delivered), and for the auth path itself (`_is_authorized_request` has no try/except so a raise denies; `_load_config` returns `{}` on corruption so a bad config denies everyone — verified clean, but on the day four security fixes landed, "swept, fails closed, here's why" is what a reader needs to distinguish from "never looked").
- **Under-counting blast radius, repeatedly:** four `resultMid` clobber sites (not two), four `_apply_kaisuuken` call sites (not three — the omitted one is the video path), five `_deleted/`-blind walks including `cmd_organize` (not four), both Loom players muted (not one), all three `ENHANCE_PLUGINS` entries dead (not two), six lightbox buttons (not three), 11+ tree walks (not seven). Three proposed fixes were mis-sized as a result, and one (`_task_image_media` for `run_generate`) would have silently dropped video recovery.
- **Sibling files went unread.** One lens indicted `docs/archive/DASHBOARD_2026-07-20.html` without opening `docs/archive/README.md`, which (a) already records the remediation and (b) *forbids* the proposed edit. Another recommended building a `--convert-existing` Panel button that the owner explicitly answered "superseded" on 2026-07-15, with the replacement shipped 2026-07-20 — the memory-flagged "don't recommend superseded designs" mistake, restated as a finding. Reading `git log` on the commit an audit's own evidence names should be routine before recommending a deletion or a build.
- **Doc-trust crept back in.** Several "verified clean" claims rest on `docs/STATE.md` saying so, in an audit whose governing rule is that the docs go stale faster than the code. They happened to be right; the method was wrong. Three cross-reference `file:line` anchors in one lens didn't resolve at all.
- **My own additions:** `moonglade_mcp.py` leaks the owner's absolute paths in a **core module's** copy-pasteable setup command (missed while another lens asserted the `docs/` script was "the only committed file with an owner-machine path"); `config.example.json` — the one file the Quickstart tells every stranger to copy — describes account management and the auth model wrongly in three clauses; the layout diagrams omit four stores of irreplaceable user-authored content; and every lens audited `loom-v2` while a fresh clone gets `master` (identical but for `docs/STATE.md` — never established by anyone).
